## 1. Введение
TaPM - это приложение для управления задачами и проектами. Продукт предназначен для помощи пользователям в организации их рабочих и личных задач, а также в управлении проектами.

Основные функции:
- Создание и управление задачами
- Организация задач по проектам
- Установка приоритетов и сроков выполнения
- Отслеживание прогресса

Продукт не будет включать в себя функции совместной работы или интеграции с внешними календарями.

## 3. Системные требования

### 3.1 Функциональные требования
1. Пользователь может создавать, редактировать и удалять задачи
2. Пользователь может создавать, редактировать и удалять проекты
3. Задачи могут быть привязаны к проектам
4. Пользователь может устанавливать приоритеты и сроки выполнения задач
5. Приложение должно отображать список задач
6. Приложение должно отображать список проектов
7. Пользователь может отмечать задачи как выполненные

### 3.2 Нефункциональные требования

#### 3.2.1 Атрибуты качества
1. Производительность: приложение должно быстро реагировать на действия пользователя (не более 1 секунды на основные операции)
2. Надежность: приложение должно сохранять данные пользователя даже при неожиданном закрытии
3. Удобство использования: интерфейс должен быть интуитивно понятным и соответствовать принципам Material Design
4. Безопасность: все данные пользователя должны храниться только локально на устройстве

### 2. Варианты использования

#### 2.1 Зарегистрировать аккаунт

**Описание**: Пользователь регистрирует новый аккаунт в системе.
- **Предусловия**: Пользователь не зарегистрирован.
- **Основной поток**:
  1. Пользователь открывает страницу регистрации.
  2. Вводит необходимые данные (e-mail, пароль, имя).
  3. Нажимает кнопку "Зарегистрироваться".
  4. Система проверяет корректность данных и создает аккаунт.
  5. Система подтверждает успешную регистрацию.
- **Альтернативные потоки**:
  1. Ошибка валидации данных — система отображает сообщение об ошибке.

#### 2.2 Войти в систему

**Описание**: Пользователь авторизуется в системе.
- **Предусловия**: Пользователь зарегистрирован.
- **Основной поток**:
  1. Пользователь открывает страницу входа.
  2. Вводит e-mail и пароль.
  3. Нажимает "Войти".
  4. Система проверяет учетные данные.
  5. Система авторизует пользователя и предоставляет доступ к функционалу.
- **Альтернативные потоки**:
  1. Неправильные данные — система отображает сообщение об ошибке.

#### 2.3 Создать проект

**Описание**: Пользователь создает новый проект.
- **Предусловия**: Пользователь авторизован.
- **Основной поток**:
  1. Пользователь нажимает кнопку "Создать проект".
  2. Вводит название проекта и описание.
  3. Нажимает "Создать".
  4. Система добавляет проект в список пользователя.
  5. Проект отображается на главной странице.

#### 2.4 Управлять проектом

**Описание**: Пользователь управляет созданным проектом (редактирует, удаляет).
- **Предусловия**: Проект существует.
- **Основной поток**:
  1. Пользователь открывает проект.
  2. Выбирает действие (редактировать или удалить).
  3. Система применяет изменения.
  
#### 2.5 Создать задачу

**Описание**: Пользователь создает задачу в рамках проекта.
- **Предусловия**: Пользователь авторизован и проект выбран.
- **Основной поток**:
  1. Пользователь нажимает кнопку "Создать задачу".
  2. Вводит описание задачи, приоритет и дедлайн.
  3. Нажимает "Создать".
  4. Система добавляет задачу к проекту.

#### 2.6 Управлять задачей

**Описание**: Пользователь редактирует или удаляет задачу.
- **Предусловия**: Задача существует.
- **Основной поток**:
  1. Пользователь открывает задачу.
  2. Выбирает действие (редактировать или удалить).
  3. Система применяет изменения.

### 3. Глоссарий

| Термин     | Описание                                                                 |
|------------|--------------------------------------------------------------------------|
| Проект     | Набор задач, связанных общей целью, который создается и управляется пользователем. |
| Задача     | Единичное действие или элемент работы в проекте. |
| Аккаунт    | Учетная запись пользователя, содержащая данные для входа и работы в системе. |
